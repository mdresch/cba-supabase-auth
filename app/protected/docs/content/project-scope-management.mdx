# Project Scope Management

Project Scope Management is a critical knowledge area that ensures the project includes all the work required, and only the work required, to complete the project successfully. It defines and controls what is and is not included in the project, helping to prevent scope creep and ensure project objectives are met.

## Key Processes

1. Plan Scope Management
2. Collect Requirements
3. Define Scope
4. Create WBS (Work Breakdown Structure)
5. Validate Scope
6. Control Scope

## Input Documents

- Project charter
- Project management plan components
- Enterprise environmental factors
- Organizational process assets
- Stakeholder register
- Requirements documentation
- Scope management plan
- Project scope statement
- Requirements traceability matrix

## Output Documents

- Scope management plan
- Requirements management plan
- Requirements documentation
- Requirements traceability matrix
- Project scope statement
- Work breakdown structure (WBS)
- WBS dictionary
- Scope baseline
- Project documents updates
- Accepted deliverables
- Work performance information
- Change requests

Effective scope management is crucial for project success, as it helps maintain focus on the project's objectives and deliverables while avoiding unnecessary work or feature creep.
